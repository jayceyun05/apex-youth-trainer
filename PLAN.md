## 1. 투 트랙 구조 ★핵심

```
시작 → [AI 전송 동의?] ─ 동의 → 트랙 A(실제 LLM, BYOK, 생성 순간만 외부 전송)
                        └ 거부 → 트랙 B(규칙 기반, 외부 전송 0, 완전 로컬)
두 트랙 공통: 무저장 · 면책 문구 · '완료/닫기' 시 즉시 wipe
```

### 1-1. 트랙 선택 안내 가이드 (시작 화면에 두 카드로 표시)

**🤖 AI 맞춤 루틴** *(인터넷 연결·정보 전송 동의 필요)*
- 👍 아이의 특이사항까지 반영해 더 유연하고 정교한 맞춤
- 👍 자연스러운 설명과 다양한 변형
- ⚠️ 생성 순간 입력 정보가 AI 서버로 전송됨 (기기엔 저장 안 됨)
- ⚠️ 인터넷 필요, 생성에 몇 초 소요
- 💡 이런 분께: "전송은 괜찮고, 최대한 정교한 맞춤을 원해요"

**🔒 오프라인 라이트 루틴** *(전송 없음·완전 비공개)*
- 👍 입력 정보가 기기 밖으로 절대 안 나감 (프라이버시 최고)
- 👍 인터넷 없이 즉시 생성, 빠름
- ⚠️ 설계된 운동 조합 기반 → AI보다 유연성·맞춤도 낮음
- ⚠️ 아주 특수한 요청은 반영 제한적
- 💡 이런 분께: "정보가 밖으로 나가는 게 꺼려져요 / 빠르고 간단하면 돼요"

## 2. 데이터 수명주기 (공통)

```
입력 → [JS 메모리 변수 1개] → 유효성 검사 3단계
   → (A) AI 전송·생성  /  (B) 로컬 규칙 생성
   → 결과 표시(상·하단 면책) → '완료/닫기' → 메모리 즉시 wipe + 입력 초기화
   → 창/탭 닫힘 → 자동 소멸 (+ beforeunload 시 명시적 wipe)
```
- localStorage / sessionStorage / 쿠키 / 파일 = **전부 미사용**
- API 키(BYOK)도 메모리에만 — 창 닫으면 소멸

## 3. 면책 문구 (확정)

- **상단 배지(항상 노출):**
  > ⚠️ AI가 만든 참고용 루틴입니다. 적용 전 전문가 확인을 권장하며, 통증·이상 시 즉시 중단하세요.
- **하단 전체 면책(고정):**
  > 본 루틴은 AI가 입력 데이터를 기반으로 생성한 참고용 가이드이며 의학적 진단·처방이 아닙니다. AI는 오류가 있을 수 있으므로, 실제 훈련에 적용하기 전 반드시 스포츠의학 전문의 또는 전문 트레이너의 확인을 받으시길 권장합니다. 통증·이상 증상이 나타나면 즉시 중단하고 전문가와 상담하세요. **본 가이드 이용에 따른 최종 판단과 책임은 보호자에게 있습니다.**
- **트랙 B(규칙 기반) 치환:** "AI가 입력 데이터를 기반으로 생성한" → "입력 데이터를 기반으로 자동 생성된", "AI는 오류가 있을 수 있으므로" → "자동 생성 결과는 부정확할 수 있으므로"

## 4. 기술 스택

- HTML + CSS + 순수 JavaScript, **단일 파일**(인라인) — 더블클릭 실행
- 저장소 없음 (메모리 변수만)
- 트랙 A: `fetch()`로 LLM API (BYOK) — 기본 `claude-sonnet-4-6`, 안전판단 강화 시 `claude-opus-4-8`
- 트랙 B: 로컬 규칙 엔진 + 종목별 운동 풀

## 5. 데이터 유효성 검사 — 3단계

### 5-1. 연령: 입력 5~100세 허용 + **15세 이하만** 관절 보호·고중량 배제 강화
### 5-2. 로직
| 단계 | 규칙 | 동작 |
|------|------|------|
| 1단계 연령 | `age<5` 또는 `age>100` | 즉시 차단 + 안내 |
| 2단계 BMI | `BMI=kg/(m²)`, 연령대 간이 임계 이탈 | 경고 |
| 3단계 투-스트라이크 | 1차: 경고+[수정]/[이대로 진행] / 강행 시: 생성 **중단** + 전문의 진단 권유 문구 강제 | 메모리 상태값 |
### 5-3. BMI 간이 임계(오타·극단 탐지용, 추후 WHO/CDC 정밀): 5~12세 11~26 / 13~17세 13~30 / 18~100세 13~45

## 11. 추가 요청 처리 (2026-06-30)

### 11-A. Beta 디자인 샘플 (claude.ai/design 링크)
- 로그인 필요한 비공개 공유 → **Claude 직접 열람 불가.**
- → 사용자가 `루틴 생성 MVP.dc.html`을 **다운로드해 첨부**하면 디자인 반영.
- 그 전까지는 **기존 첨부 `index.html` 스타일**을 출발점으로 사용.

### 11-B. 외부 스포츠/신체 데이터 사이트
- 실시간 크롤링·API: ① 무저장/프론트 철학과 충돌 ② 다수 로그인·API키·동적페이지 ③ CORS → **1차 부적합.**
- → 방식: 필요한 **벤치마크 수치만 정적 데이터로 내장**(코드 내 `benchmarks` 객체).
- → 수치 정확성: **추정 금지**(CLAUDE.md). 공개 확정 수치만 사용, 불확실하면 사용자 확인/CSV 제공 후 반영.
- **출처 우선순위(1차):**
  1. 신체 성장(키/몸무게 백분위): 질병관리청 소아청소년 성장도표 / 미 CDC Growth Charts
  2. 체력(악력·매달리기·달리기): 국민체력100 / PAPS
  3. 야구 구속(연령별): Driveline Youth Velocity / Perfect Game 9U·10U / MLB PitchSmart
- **자동 수집 후보:** 공공데이터포털(data.go.kr) Open API, The Open Biomechanics Project CSV — *R/Python 별도 수집 스크립트로 처리, 앱과 분리.*

### 11-C. 또래 비교 대시보드 (마스터 프롬프트)
- 신규 기능: 아이 데이터 vs 국내외 또래 평균 비교 시각화.
- ASCII 막대그래프 → 웹앱이므로 **실제 HTML/CSS 막대그래프 권장**(가독성↑).
- 구성: ① 신체 스펙 ② 야구 구속 ③ 기초 체력/코어 ④ 종합 총평. 추정치는 "근사 추정" 명시.
- 위치: 루틴 결과와 별개 **'또래 비교 리포트'** 화면.
- → **Phase 4(데이터 내장) → Phase 5(대시보드)** 로 진행. 1차 루틴 베타 이후 권장.
